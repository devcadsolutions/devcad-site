# Site Memory Maintenance Skill

Use this skill when updating project memory after important design, architecture, deployment, or content changes.

## Goal

Keep future agent work grounded in the current reality of the site.

## Update When

- deployment behavior changes
- navigation structure changes
- major page layouts are redesigned
- branding, assets, or profile imagery change
- important constraints are discovered

## Rules

- Update the smallest relevant file in `agent/memory/`.
- Prefer bullets over paragraphs.
- Keep facts concrete and repo-specific.
- Remove stale notes instead of accumulating contradictions.

## Current Memory Files

- `agent/memory/project-memory.md`
- `agent/memory/recent-changes.md`
